# FroG Figma Plugin 🐸

A delightful productivity and break-reminder companion for Figma and FigJam. **FroG** sits on your canvas, keeps you company with smooth hand-drawn animations while you type/focus, and guides you through healthy break cycles (drinking water and stretching).

---

## 🌟 Features

- **Onboarding Experience**: 
  - **Step 1**: A cute introduction dialogue bubble (`ask-01.png`) appears. Click anywhere on the frog to proceed.
  - **Step 2**: A configuration dialogue bubble (`ask-02.png`) appears along with a slider to set your focus timer.
  - Onboarding Step 1 is only shown once and is saved locally to Figma's `clientStorage`.
- **Delightful Focus Timer**:
  - Drag the custom hand-drawn slider or use the `-` / `+` buttons to snap-select a timer duration from **10 to 60 minutes** (in 5-minute intervals).
  - While running, the frog works hard typing on their laptop.
  - The action button dynamically updates to show the remaining time (`mm:ss`).
- **Easy Pause & Resume**:
  - Click the countdown timer button at any time to pause the countdown.
  - While paused, you can easily tweak the time on the slider or click `START` to resume right where you left off.
- **Automated Break Cycle**:
  - When the countdown reaches `00:00`, the break cycle triggers.
  - **Drinking Break**: The frog plays a watering/drinking animation.
  - **Stretching Break**: Automatically transitions into a stretching routine, pausing halfway.
  - **Done & Reset**: Click the `DONE` action button to complete the stretch. The frog sits back down and automatically starts a new focus cycle with the previously configured time.
- **Custom Hand-Drawn UI System**:
  - Entirely built using hand-drawn PNG assets for the buttons, slider tracks, thumbs, plus/minus controls, and dialogue overlays.
  - Bypasses Figma's automatic dark/light theme color inverting to maintain its beautiful hand-drawn sketch style consistently.

## ⚙️ Building the UI

The frontend `ui.html` is generated by `build.js`. It reads raw image assets (the frog animation frames from the sibling directory `../figma-froggy/FroQ/widget-src/frames.json` and local controls), base64-encodes them, and embeds them directly into a single file to bypass network request limits.

To recompile `ui.html` after making changes to style, structure, or assets:

```bash
node build.js
```

## 🚀 Installation & Local Development

### 1. Import to Figma
1. Open the **Figma Desktop App**.
2. Click the Figma menu icon, then go to **Plugins** ➔ **Development** ➔ **Import plugin from manifest...**.
3. Choose the [manifest.json](file:///Users/dzcat/Library/CloudStorage/Dropbox/Works/www/tools/figma-FroG-plugin/manifest.json) file inside this directory.

### 2. Run the Plugin
1. Open any Figma design file or FigJam board.
2. Right-click the canvas and select **Plugins** ➔ **Development** ➔ **FroG**.
3. To reload changes while developing: right-click the plugin UI panel and choose **Reload plugin** (or press `Cmd + Option + P` / `Ctrl + Alt + P` to run the last plugin again).
